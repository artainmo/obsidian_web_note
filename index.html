Obsidian LLM

Create a chatbot that can answer questions based on information from an obsidian vault.

Create an LLM that can be fine tuned with own notes. This could be done with an AWS ML service or by fine-tuning with labeled data an Hugging Face Model. See AWS course and last machine learning course.
Amazon Kendra could be used to find relevant responses to natural language questions in repositories of text.

This project could allow the creation of such a fine-tuned model: AI, machine-learning, startup-idea.

Maybe the place where you query the AI could be a home page. On that home page you would have one place to query but also a listing of all the tags. AI could be used to group the tags in different categories.
However, the current version of obsidian uses constant side-bar menus as a replacement of a home page. Thus, maybe I should align with that. Put an 'ask' icon on the top bar to open the AI search page. The tag list in the right side-bar could be displayed differently as another project.

Use the AWS Titan LLM to perform transfer learning on. Finish the course I got from 19, to do the AWS Titan part. Ask Edouard if AWS Titan is worth it.

However, NotebookLM already exists and does something similar. Maybe I could use its API?

Maybe combine Amazon Kendra with an LLM. First use Amazon Kendra to show where in the vault the question answers could lie. Subsequently, use an LLM to take those answers and synthesize a conclusion from it.

RAG in the context of LLMs takes texts initially, and answers a subsequent question based on that. This is an algorithm I could use. I can ask Arthur Vandecan about it.
Not only Arthur Vandecan, Edouard also confirmed me that RAG is the algorithm I should use. RAG would even be better than Amazon Kendra (because Kendra uses a simple search engine), while creating a RAG locally is free. Cohere integrates RAGs for other enterprises.
